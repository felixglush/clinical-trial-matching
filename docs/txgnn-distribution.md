# TxGNN distribution format

**Research date:** 2026-05-21. **Re-verify before re-running the build if more than a few months have passed** — repo layouts and Dataverse paths change.

## Summary

**TxGNN does not publicly distribute a pre-computed bulk prediction table.** The Zitnik lab provides:

1. **txgnn.org** — interactive per-query web explorer (no bulk export).
2. **github.com/mims-harvard/TxGNN** — Python package; predictions must be generated by running the model's `.predict()` API. GPU required for full coverage.
3. **Harvard Dataverse `DOI:10.7910/DVN/IXA7BM`** — PrimeKG only (the underlying graph), not predictions.
4. **Nature Medicine 2024 supplementary** — benchmark and FDA-validation tables only, not a full disease × drug matrix.

The implication for this project: live TxGNN inference is out of scope (no model hosting per spec), so we cannot generate the dump ourselves. We commit a **hand-curated starter dataset** in the same JSON shape the runtime expects. Real TxGNN inference output can replace it later without code changes.

## Source

- Web explorer (per-query): `http://txgnn.org`
- Code (model + training): `https://github.com/mims-harvard/TxGNN`
- Underlying KG (PrimeKG): `https://dataverse.harvard.edu/dataset.xhtml?persistentId=doi:10.7910/DVN/IXA7BM`
- Paper: Nature Medicine 2024 (Huang/Zitnik), `https://www.nature.com/articles/s41591-024-03233-x`
- License: unverified — needs manual Dataverse click-through. Treat as restrictive until confirmed.

## File format (when generating yourself via the TxGNN model)

- TxGNN uses **PrimeKG internal node ids** for both diseases and drugs — *not* MONDO and *not* DrugBank.
- To map back to MONDO for our SNOMED→MONDO crosswalk: join against PrimeKG `nodes.csv` (`node_source` column carries `MONDO` for disease nodes).
- Output schema (inferred from API + paper, unverified at column level):
  - Disease id column: PrimeKG node id
  - Drug id column: PrimeKG node id
  - Drug name: not in row; join against PrimeKG `nodes.csv`
  - Indication probability: 0–1 scalar (likely `indication_score`)
  - Contraindication probability: 0–1 scalar (likely `contraindication_score`)
- Explainer paths: **not pre-materialized**. Generated on demand by TxGNN's Explainer module per query.

## Our JSON shape (independent of source)

We pick our own shape, keyed by MONDO id (so it composes with the existing SNOMED→PrimeKG crosswalk):

```jsonc
// apps/agent/src/data/txgnn-predictions.json
{
  "MONDO:0005148": [
    { "drugId": "DB00331", "drugName": "metformin", "predIndication": 0.94, "predContraindication": 0.08 }
  ]
}

// apps/agent/src/data/txgnn-explanations.json
{
  "MONDO:0005148::DB00331": {
    "nodes": [...],
    "edges": [...]
  }
}
```

The build script (`scripts/build-txgnn-data.ts`) consumes inputs and emits these JSONs. Today the input is a hand-curated TSV under `data/txgnn-curated/`. When a real bulk source becomes available (or someone runs the TxGNN model locally), swap the parser and the rest of the pipeline is unchanged.

## Coverage scope of the starter dataset

We cover the MONDO ids associated with the four Synthea archetype patients used elsewhere in the project (see `apps/agent/src/tools/snomed-mondo.test.ts`):

- `MONDO:0008383` — rheumatoid arthritis
- `MONDO:0005148` — type 2 diabetes mellitus
- `MONDO:0005233` — non-small cell lung carcinoma (closest PrimeKG match)
- `MONDO:0007254` — breast cancer
- `MONDO:0005300` — chronic kidney disease

Roughly 5–10 well-known drugs per disease, with realistic but **clinically curated** probability scores. Mark this as not-the-paper data anywhere it surfaces.
